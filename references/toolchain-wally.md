# Toolchain: Wally, linting, formatting, editor, Git

Load this for package management, code quality tooling, editor setup, the sourcemap, or
`.gitignore`. These are what make filesystem-based Roblox development pleasant and safe.

## The supporting cast

| Tool | Job | Installed via |
|---|---|---|
| **Wally** | Package manager | Rokit / Aftman |
| **StyLua** | Formatter (Prettier for Luau) | Rokit / Aftman |
| **Selene** | Linter / static analysis | Rokit / Aftman |
| **Luau LSP** | Autocomplete, types, go-to-def (VS Code) | VS Code marketplace |
| **Lune** | Run Luau outside Roblox (tests, build scripts) | Rokit / Aftman |

Pin them all in `rokit.toml` (or `aftman.toml`) so the whole team and CI match. See
`references/rojo-setup.md` for the toolchain-manager basics.

## Wally: packages

Dependencies are declared in `wally.toml` and installed into a `Packages/` folder, which the
project maps to `ReplicatedStorage.Packages`.

```toml
[package]
name = "yourname/my-game"
version = "0.1.0"
registry = "https://github.com/UpliftGames/wally-index"
realm = "shared"

[dependencies]
Knit = "sleitnick/knit@1.7.0"
Signal = "sleitnick/signal@2.0.1"
Trove = "sleitnick/trove@1.4.0"
Janitor = "howmanysmall/janitor@1.18.3"
TableUtil = "sleitnick/table-util@1.2.1"
Roact = "roblox/roact@1.4.4"
Rodux = "roblox/rodux@3.0.0"
RoactRodux = "roblox/roact-rodux@0.3.0"
Promise = "evaera/promise@4.0.0"

[dev-dependencies]
TestEZ = "roblox/testez@0.4.1"
```

```bash
wally install        # resolve + download into Packages/
wally search promise # find a package
```

> The framework this skill is based on used a private registry for some in-house modules
> (animation, sound, VFX, funnels). For a public/open project, default to the public
> UpliftGames registry and the public equivalents. ProfileService is often vendored into
> `src/server/Modules` rather than pulled from Wally, that's a normal choice.

**Realms** decide where a package lands: `shared` → ReplicatedStorage (both sides),
`server` → ServerScriptService (server only), `dev` → development only.

After installing, regenerate the sourcemap so the LSP can resolve package types:

```bash
rojo sourcemap default.project.json -o sourcemap.json
```

## StyLua: formatting

Stop arguing about style in code review. `stylua.toml`:

```toml
column_width = 120
line_endings = "Unix"
indent_type = "Tabs"
indent_width = 4
quote_style = "AutoPreferDouble"
call_parentheses = "Always"
```

```bash
stylua src/            # format everything
stylua --check src/    # CI: fail if anything is unformatted
```

Format-on-save is worth setting up (VS Code config below).

## Selene: linting

Catches unused variables, shadowing, bad API use, and more before runtime. `selene.toml`:

```toml
std = "roblox"

[rules]
unused_variable = "warn"
shadowing = "warn"
undefined_variable = "deny"
multiple_statements = "warn"
```

```bash
selene src/
selene generate-roblox-std   # refresh the Roblox global definitions
```

The `std = "roblox"` line is what stops `game`, `workspace`, etc. from being flagged as
undefined.

## Editor: VS Code

The two extensions that matter most: **Luau LSP** (`JohnnyMorganz.luau-lsp`) and **Rojo**
(`evaera.vscode-rojo`). Add Selene and StyLua extensions for inline warnings and
format-on-save.

`.vscode/settings.json`:

```json
{
  "luau-lsp.sourcemap.enabled": true,
  "luau-lsp.sourcemap.rojoProjectFile": "default.project.json",
  "luau-lsp.sourcemap.autogenerate": true,
  "luau-lsp.types.roblox": true,
  "[luau]": {
    "editor.defaultFormatter": "JohnnyMorganz.stylua",
    "editor.formatOnSave": true,
    "editor.insertSpaces": false
  },
  "files.associations": { "*.luau": "luau", "*.lua": "luau" }
}
```

`.vscode/extensions.json`:

```json
{
  "recommendations": [
    "JohnnyMorganz.luau-lsp",
    "evaera.vscode-rojo",
    "Kampfkarren.selene-vscode",
    "JohnnyMorganz.stylua"
  ]
}
```

If Roblox APIs show red squiggles, the sourcemap isn't generating, run the `rojo sourcemap`
command manually and confirm `luau-lsp.types.roblox` is true.

## Git: what to commit and ignore

`.gitignore`:

```gitignore
# Built place/model files (regenerated by Rojo)
*.rbxlx
*.rbxl
*.rbxm
*.rbxmx
# allow a committed base place if you use one:
# !base.rbxlx

# Wally packages — installed, not committed
/Packages/

# Auto-generated
sourcemap.json
roblox.yml

# OS / editor
.DS_Store
Thumbs.db
.vscode/*
!.vscode/settings.json
!.vscode/extensions.json
```

**Do** commit: `default.project.json`, `wally.toml`, `wally.lock` (reproducible installs),
`rokit.toml`/`aftman.toml` (matching tool versions), `stylua.toml`, `selene.toml`, and all of
`src/`.

**Don't** commit: `Packages/` (run `wally install` instead) or build artifacts (`*.rbxl*`).

## Studio + Git coexistence

Two-way sync handles script *content* well, but not arbitrary instance/level edits. The
common arrangement: code lives on the filesystem (source of truth, synced via Rojo); maps and
geometry are authored in Studio and captured by committing a `base.rbxlx` place that Rojo
builds on top of:

```bash
rojo build default.project.json --output game.rbxlx --base base.rbxlx
```

## CI (optional but recommended)

A minimal GitHub Actions pipeline runs format-check, lint, and a build on every PR:

```yaml
name: CI
on: [push, pull_request]
jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: CompeyDev/setup-rokit@v0.1.2   # or setup-aftman
      - run: stylua --check src/
      - run: selene src/
      - run: wally install
      - run: rojo build default.project.json -o build.rbxl
```

Action names/versions drift, check the marketplace before copying. The shape (install
toolchain → check format → lint → install packages → build) is the stable part.
